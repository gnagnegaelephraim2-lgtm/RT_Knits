# Plan: Restructure Auth & Fix Field Mapping Chaos

## Problem Summary

The project has **two divergent codebases** — `app.ts` (TypeScript source) and `app.js` (compiled JS) — that drifted apart during dozens of incremental fixes. The core issues:

1. **Interfaces don't match DB schema** — `Department.id` vs `department.department_id`, `Asset.code` vs `asset.asset_code`, etc.
2. **syncWithSupabase maps fields to wrong names** — creates objects with `code`/`id` but lookup caches expect `asset_code`/`department_id`
3. **Signup omits `pin_hash`** in the Supabase write path — new users can't log in
4. **Login has a security hole** — Supabase direct path skips PIN validation entirely
5. **`selectedDeptId = departments[0].department_id`** crashes on empty array at page load
6. **`asset_id` is dropped during sync** — all task-to-asset lookups break
7. **No NITA API auth endpoint** — login repurposes `api-admin-read` which exposes raw user data

## Strategy: Single Source of Truth

**Kill the TypeScript file.** `app.ts` is never compiled — `app.js` is hand-edited. Having two sources that drift is the root cause of every bug. We maintain `app.js` directly, the single source of truth.

**Use DB column names directly** in JS objects. No mapping layer. `department.department_id`, `asset.asset_code`, `technician.technician_id`, `technician.full_name`. This eliminates an entire class of bugs.

### Step 2: Fix rebuildLookupCaches to use DB column names

```js
const assetMap = new Map();    // keyed by asset_code (string like "39")
const assetIdMap = new Map();  // keyed by asset_id (UUID)
const deptMap = new Map();     // keyed by department_id (UUID)

function rebuildLookupCaches() {
  assetMap.clear();
  assetIdMap.clear();
  deptMap.clear();
  for (const a of assets) {
    assetMap.set(a.asset_code, a);
    if (a.asset_id) assetIdMap.set(a.asset_id, a);
  }
  for (const d of departments) deptMap.set(d.department_id, d);
}
```

### Step 5: Rewrite login handler — clean 2-step cascade

```js
// Step 1: NITA API (primary)
try {
  const result = await NITA_API.adminRead('app_user', 'phone_number', phoneInput, 1);
  if (result.rows?.length > 0) {
    const user = result.rows[0];
    if (user.pin_hash && user.pin_hash !== pinHash) {
      alert("Incorrect PIN.");
      return;
    }
    const role = user.role === 'admin' ? 'coordinator' : user.role;
    matchedUser = { name: user.full_name, phone: phoneInput, role, user_id: user.user_id };
  }
} catch {}

// Step 2: Local fallback (localStorage custom users)
if (!matchedUser) {
  const custom = JSON.parse(localStorage.getItem("nita_custom_users") || "{}")[phoneInput];
  if (custom && custom.pinHash === pinHash) {
    matchedUser = { name: custom.name, phone: phoneInput, role: custom.role, user_id: generateId('user') };
  }
}
```

Remove Rust server auth (not deployed) and Supabase direct auth (security hole).
